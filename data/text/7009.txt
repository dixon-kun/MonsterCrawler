        #  Core Infrastructure Roles / Broad Description
        * This role will work in the public cloud team developing secure services and platforms on public cloud providers for consumption by JPMC internal teams\.
        * While the role will be broadly architectural, the role will require a broad working knowledge of infrastructure, security processes and technical implementation and code development\.
        * Every service delivered by this team will be built with security from the ground up\. This role will need an understanding of security guidelines and regulation and be able to map these to both process and technical solutions to deliver end to end security\.
        * Repeatable processes will be a key part of delivering the standardised and scalable cloud infrastructure we require for this project\. Cloud environment are infrastructure delivered from code and this role will deliver a range of template and code based delivery tools to automate every aspect of the environments we wish to deploy\.
        * The skills spelled out below are general skills / background required
        ## Soft Skills
        * The successful candidate must be an experienced technologist with a natural curiosity for new processes and technology and enthusiasm for delivering new services\. This must be balanced with a realistic understanding of the requirements of managing sensitive data within a highly regulated environment\. Experience of working with public cloud is advantageous but not essential\.
        * As part of a new team they must be a motivated self\-starter with the ability to focus and deliver to tight deadlines and manage multiple areas of work simultaneously\.
        * The project will involve working with a wide range of stakeholders with contrasting interests\.
        * A key skill for this role will be working with differing groups to drive conversations constructively, establish requirements and the basis for these requirements then work to product solutions and architectures which meet the agreed requirements\.
        ## Architecture
        * Experience of delivering architectures for platforms and architectural standards is a key requirement\. Cloud platforms are, by their nature, open to experimentation and the flexibility to propose architectural ideas and models test on the target environment and refine the architecture based on results and observations will be a key skill\.
        * The successful candidate will have a good understanding of technical architecture and enterprise architecture\. Understanding of ITIL and TOGAF / Zachman frameworks are advantageous\. Equally experience of Agile and Continuous Delivery / Integration are also highly desirable\.
        ## Infrastructure
        * A high level understanding of networking concept including CIDR blocks, route tables, firewall rules, IP port management, forwarding and NAT, and WAN and LAN design is required\. Detailed knowledge of BGP, protocol stacks, and TCP / IP communication would be desirable but is not essential\.
        * You will need a good user level understanding of the operation and build of a standard Linux server or Windows box\. This is to include the use of typical user level commands, the ability to compile and install common packages and use common tools such as network and performance testing utilities\.
        ##  Development
        * Ability to program in Java or Python or other languages with a Cloud Providers CLI is highly desirable\. Experience with the Boto libraries for Python or AWS Tools for PowerShell would be very useful but not essential\.
        * Understanding of XML based template markup languages is desirable\. AWS CloudFormation is a desirable skill but not required\.
        * Experience with automation tools such as GitHub, Jenkins, Ansible, Puppet, Chef, Salt or Opsworks are all useful skills\.
        ## General Skills
        * Experience of working in an international team for a major enterprise is desirable\.
        * Experience of both Waterfall and Agile project management methodologies\.
        * Ability to write technical and API documentation and project reports tailored to a range of different audiences\.
        * Ability to deliver technical talks and presentation and be comfortable speaking to audiences ranging from technical developers to senior stakeholders\. Understanding how to deliver these presentation via video conference, WebEx and in person\.
        * Comfortable working in high pressure situations and understand how to ask for help and manage conflicting priorities\.
